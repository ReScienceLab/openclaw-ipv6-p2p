# Ralph Progress Log

## Codebase Patterns
- In `gateway/server.mjs`, keep peer-protocol response payloads backward compatible even when internal registry helpers are renamed; `/peer/peers` and `/peer/announce` still emit `peers`.
- Gateway registry persistence lives in `gateway/server.mjs` as a self-contained `$DATA_DIR/registry.json` helper: load on startup, filter out entries older than `STALE_TTL_MS`, and write atomically through `registry.json.tmp` + rename.
- In `gateway/server.mjs`, registry persistence side effects should stay explicit: announce-driven `upsertAgent()` calls opt into debounced saves, while prune/shutdown paths use `flushRegistry()` for immediate durability.
- In `gateway/server.mjs`, `/peer/message` is relay-only: it should verify and forward `world.state` payloads, but registry membership must come only from `/peer/announce`.
- In `gateway/server.mjs`, lifecycle cleanup should retain server and timer handles in module scope so signal handlers can flush the registry once, clear intervals, and close listeners idempotently.
- In `gateway/server.mjs`, `/health` must keep `ok: true` for compatibility while deriving `status` from registry contents: `ready` when `worlds > 0`, `warming` when `agents > 0` but no worlds, and `empty` when the registry is empty.
- In `packages/agent-world-sdk/src/world-server.ts`, active world membership is tracked by `agentLastSeen` plus `agentEndpoints`; `getMembers()` already treats membership as their intersection, while `peerDb` is broader discovery state and should not be used as the source of truth for joined-world recipients.
- In `packages/agent-world-sdk/src/world-server.ts`, broadcast delivery must use the selected member's own `agentEndpoints` entry; do not fall back to `peerDb` for per-recipient endpoint lists after membership filtering.
- In `packages/agent-world-sdk/src/world-server.ts`, world-state broadcasts should attempt every endpoint registered on an active member, not stop after the first successful `/peer/message` delivery.
- Full repo validation currently needs both TypeScript builds before `node --test test/*.test.mjs`: run `npm run build` and `npm --prefix packages/agent-world-sdk run build` so both `dist/` trees exist for the tests.
- In `test/world-state-broadcast.test.mjs`, model a discovered-but-not-joined peer by sending a signed `/peer/announce` without `world.join`; that seeds the old `peerDb` shape while leaving `agentLastSeen` and `agentEndpoints` empty for exclusion coverage.
- In `test/world-state-broadcast.test.mjs`, prove endpoint ownership by seeding both a discovered non-member and a joined member, intercepting outbound `/peer/message` calls, and asserting the exact hit-port set matches only the joined member's endpoints.
