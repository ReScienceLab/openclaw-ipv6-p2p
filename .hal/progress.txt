- For the `compound/fix-sdk-protocol-consistency-bugs` branch, keep non-bookkeeping changes scoped to `packages/agent-world-sdk/src/world-server.ts`, `packages/agent-world-sdk/src/world-ledger.ts`, `packages/agent-world-sdk/src/peer-protocol.ts`, and only the minimal regression files needed to verify those three fixes.
- In `packages/agent-world-sdk/src/world-server.ts`, `world.state` broadcasts still go through `/peer/message`, so their signatures must use `DOMAIN_SEPARATORS.MESSAGE` rather than `DOMAIN_SEPARATORS.WORLD_STATE`.
- In `packages/agent-world-sdk/src/world-ledger.ts`, define `LEDGER_SEPARATOR` directly as ``AgentWorld-Ledger-${PROTOCOL_VERSION}\0`` from `./version.js` instead of deriving it from `DOMAIN_SEPARATORS.MESSAGE`.
- For private SDK constants that are not exported, prefer a root `node:test` regression against the built `packages/agent-world-sdk/dist` artifact plus a narrow runtime verification instead of widening the public API only for tests.
- For direct SDK handler regressions that the integrated HTTP suites do not hit, prefer a root `node:test` file that imports built `packages/agent-world-sdk/dist` entrypoints and exercises `registerPeerRoutes()` with `fastify.inject()` instead of starting a network listener.
- For direct SDK `/peer/message` route regressions, use a body-signed `DOMAIN_SEPARATORS.MESSAGE` payload with `fastify.inject()` to reach the callback branch without depending on HTTP header-signing or a live listener.
- In `packages/agent-world-sdk/src/peer-protocol.ts`, callback-style `reply.send(...)` branches inside `registerPeerRoutes()` should explicitly `return reply.send(...)` so custom responders do not leave the async handler control flow ambiguous.
- For deterministic identity-helper regressions in this branch, use `nacl.sign.keyPair.fromSeed(Buffer.alloc(32, 1))`; its public key is `iojj3XQJ8ZX9UtstPLpdcspnCb8dlBIb83SIAbQPb1w=` and it maps to stable literals for both `deriveDidKey()` and `toPublicKeyMultibase()`.
- For Base58 zero-boundary fixes in this repo, both codec halves need explicit all-zero handling: the encoder must not append the synthetic zero digit after leading `1`s, and the decoder must not retain the synthetic accumulator byte for all-leading-`1` inputs.
- Direct Base58 boundary regressions in `test/base58.test.mjs` should import compiled helpers from the SDK `dist/` split points: `base58Encode` from `dist/identity.js` and `base58Decode` from `dist/peer-protocol.js`.
- In `packages/agent-world-sdk/src/peer-protocol.ts`, `/peer/key-rotation` validation is authoritative inside `registerPeerRoutes()`, and new key-binding checks should run after multibase decoding but before `peerDb.upsert(...)`.
- Reuse `agentIdFromPublicKey()` from `packages/agent-world-sdk/src/crypto.ts` for any public-key-to-agent binding check; do not duplicate the derivation logic in route handlers or tests.
- Repo-level key-rotation tests exercise `dist/peer-server.js`, so whenever the SDK `registerPeerRoutes()` rotation contract changes, mirror the same validation and error payload in `src/peer-server.ts` before relying on `test/key-rotation.test.mjs`.
- In `gateway/server.mjs`, keep peer-protocol response payloads backward compatible even when internal registry helpers are renamed; `/peer/peers` and `/peer/announce` still emit `peers`.
- Gateway registry persistence lives in `gateway/server.mjs` as a self-contained `$DATA_DIR/registry.json` helper: load on startup, filter out entries older than `STALE_TTL_MS`, and write atomically through `registry.json.tmp` + rename.
- In `gateway/server.mjs`, registry persistence side effects should stay explicit: announce-driven `upsertAgent()` calls opt into debounced saves, while prune/shutdown paths use `flushRegistry()` for immediate durability.
- In `gateway/server.mjs`, `/peer/message` is relay-only: it should verify and forward `world.state` payloads, but registry membership must come only from `/peer/announce`.
- In `gateway/server.mjs`, lifecycle cleanup should retain server and timer handles in module scope so signal handlers can flush the registry once, clear intervals, and close listeners idempotently.
- In `gateway/server.mjs`, `/health` must keep `ok: true` for compatibility while deriving `status` from registry contents: `ready` when `worlds > 0`, `warming` when `agents > 0` but no worlds, and `empty` when the registry is empty.
- In `packages/agent-world-sdk/src/world-server.ts`, active world membership is tracked by `agentLastSeen` plus `agentEndpoints`; `getMembers()` already treats membership as their intersection, while `peerDb` is broader discovery state and should not be used as the source of truth for joined-world recipients.
- In `packages/agent-world-sdk/src/world-server.ts`, broadcast delivery must use the selected member's own `agentEndpoints` entry; do not fall back to `peerDb` for per-recipient endpoint lists after membership filtering.
- In `packages/agent-world-sdk/src/world-server.ts`, world-state broadcasts should attempt every endpoint registered on an active member, not stop after the first successful `/peer/message` delivery.
- Full repo validation currently needs both TypeScript builds before `node --test test/*.test.mjs`: run `npm run build` and `npm --prefix packages/agent-world-sdk run build` so both `dist/` trees exist for the tests.
- The SDK Base58 codec is split across files: `packages/agent-world-sdk/src/identity.ts` owns `base58Encode` for DID/multibase generation, while `packages/agent-world-sdk/src/peer-protocol.ts` owns `base58Decode` for multibase parsing, so leading-zero fixes need paired coverage on both sides.
- In `test/world-state-broadcast.test.mjs`, model a discovered-but-not-joined peer by sending a signed `/peer/announce` without `world.join`; that seeds the old `peerDb` shape while leaving `agentLastSeen` and `agentEndpoints` empty for exclusion coverage.
- In `test/world-state-broadcast.test.mjs`, prove endpoint ownership by seeding both a discovered non-member and a joined member, intercepting outbound `/peer/message` calls, and asserting the exact hit-port set matches only the joined member's endpoints.
- In `test/world-members.test.mjs`, the suite reuses one world server across subtests, so membership assertions should verify inclusion or exclusion of specific agent IDs instead of assuming the member list starts empty.
